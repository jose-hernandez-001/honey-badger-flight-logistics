REST API
========

The REST API is defined in ``src/main/resources/static/openapi.yaml`` and
follows OpenAPI 3.0.  Spring MVC controller code is generated by
``openapi-generator-maven-plugin``; only the delegate implementations are
hand-written.

Interactive Documentation
--------------------------

Two API browsers are bundled and served as static pages:

.. list-table::
   :header-rows: 1
   :widths: 20 80

   * - URL
     - Description
   * - ``/scalar.html``
     - Scalar API reference (modern, recommended).
   * - ``/redoc.html``
     - ReDoc API reference.
   * - ``/swagger-ui.html``
     - Swagger UI (served by Springdoc).

Base URL
--------

All API endpoints are prefixed with ``/api/v1``.

Routes
--------

.. list-table::
   :header-rows: 1
   :widths: 10 30 60

   * - Method
     - Path
     - Description
   * - ``GET``
     - ``/api/v1/routes``
     - List routes. Supports ``status``, ``page``, ``size``, and ``sort``
       query parameters.
   * - ``POST``
     - ``/api/v1/routes``
     - Create a route. Body: ``RouteRequest``.
   * - ``GET``
     - ``/api/v1/routes/{routeId}``
     - Retrieve a single route by UUID.
   * - ``PUT``
     - ``/api/v1/routes/{routeId}``
     - Fully replace a route. Body: ``RouteRequest``.
   * - ``PATCH``
     - ``/api/v1/routes/{routeId}``
     - Partially update a route. Body: ``RoutePatch``.
   * - ``DELETE``
     - ``/api/v1/routes/{routeId}``
     - Delete a route and all its waypoints.

Waypoints
---------

All waypoint endpoints are nested under their parent route.

.. list-table::
   :header-rows: 1
   :widths: 10 45 45

   * - Method
     - Path
     - Description
   * - ``GET``
     - ``/api/v1/routes/{routeId}/waypoints``
     - List waypoints for a route (paginated).
   * - ``POST``
     - ``/api/v1/routes/{routeId}/waypoints``
     - Add a waypoint to a route.
   * - ``GET``
     - ``/api/v1/routes/{routeId}/waypoints/{waypointId}``
     - Retrieve a single waypoint.
   * - ``PUT``
     - ``/api/v1/routes/{routeId}/waypoints/{waypointId}``
     - Fully replace a waypoint.
   * - ``PATCH``
     - ``/api/v1/routes/{routeId}/waypoints/{waypointId}``
     - Partially update a waypoint.
   * - ``DELETE``
     - ``/api/v1/routes/{routeId}/waypoints/{waypointId}``
     - Remove a waypoint from a route.
   * - ``PUT``
     - ``/api/v1/routes/{routeId}/waypoints/reorder``
     - Reorder all waypoints by supplying an ordered list of UUIDs.

Route Status
--------------

The ``status`` field accepts the following lifecycle values:

.. list-table::
   :header-rows: 1
   :widths: 25 75

   * - Value
     - Description
   * - ``PLANNING``
     - Route is being planned; waypoints may be added or reordered.
   * - ``IN_PROGRESS``
     - Route is active; the aircraft is currently flying the route.
   * - ``COMPLETED``
     - Route has been successfully completed.
   * - ``CANCELLED``
     - Route was cancelled before or during execution.

gRPC API
--------

Protobuf service definitions are generated from the same OpenAPI spec during
the ``initialize`` build phase and emitted to
``target/generated-sources/proto/``.  The compiled stubs are served by
Spring gRPC using the auto-configured ``grpc.server.port`` (default ``9090``).

Error Responses
---------------

All error responses conform to `RFC 9457 Problem Details
<https://www.rfc-editor.org/rfc/rfc9457>`_ and include at minimum ``type``,
``title``, ``status``, and ``detail`` fields.
